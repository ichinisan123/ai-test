A **programming language** is a formal system of rules and syntax used to write instructions that a computer can understand and execute. It allows humans to communicate with machines by writing code to create software, applications, scripts, and more.

### Key Features of Programming Languages:
1. **Syntax** – The set of rules defining how code must be structured (e.g., semicolons, brackets, indentation).
2. **Semantics** – The meaning behind the code (what the instructions actually do).
3. **Abstraction** – Hiding complex details to simplify coding (e.g., functions, classes).
4. **Variables & Data Types** – Used to store and manipulate data (e.g., numbers, text, booleans).
5. **Control Structures** – Loops (`for`, `while`), conditionals (`if-else`), and logic to control program flow.
6. **Libraries & Frameworks** – Pre-written code to extend functionality (e.g., Python’s `NumPy`, JavaScript’s `React`).

### Types of Programming Languages:
1. **Low-Level Languages**  
   - Close to machine code (e.g., **Assembly**).  
   - Fast but hard to write.  

2. **High-Level Languages**  
   - Easier for humans (e.g., **Python, Java, C++**).  
   - Require compilation/interpretation to run.  

3. **Scripting Languages**  
   - Interpreted, often for automation (e.g., **JavaScript, Python, Bash**).  

4. **Functional vs. Imperative**  
   - **Imperative (C, Java):** Step-by-step instructions.  
   - **Functional (Haskell, Lisp):** Focus on mathematical functions.  

5. **Object-Oriented (OOP)**  
   - Organizes code into objects (e.g., **Java, C#, Python**).  

### Examples:
- **Python** – Easy syntax, great for beginners and AI.  
- **JavaScript** – Powers web browsers.  
- **C/C++** – Used for system programming, games.  
- **SQL** – Manages databases.  

### How Code Runs:
- **Compiled (C, Go):** Translated to machine code before execution.  
- **Interpreted (Python, Ruby):** Executed line-by-line by an interpreter.
